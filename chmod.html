Linux Command: chmod

Description:

The chmod command is used to change the file permissions of a file or directory. It can modify who can read, write, or execute a file.

Purpose & Usage:

Purpose: To manage file permissions (who can access a file and how).

Usage: Often used by system administrators to secure files or allow access.

Examples:

Grant execute permission to a script:

chmod +x script.sh

This command grants execute permission to the script script.sh.

Change file permissions to read/write for owner, and read-only for others:

chmod 644 file.txt

This command sets the permissions for the file file.txt to allow read/write access for the owner and read-only access for others.

Explanation:

Common Use Case: Granting execution permissions or controlling access to files.

Importance: Crucial for securing files and directories in multi-user environments, allowing administrators to control access levels to sensitive files.
